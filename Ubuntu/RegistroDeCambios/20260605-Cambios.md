# Cambios 2026-06-05

## Rol `clienteNAS` — fallo en re-ejecución: "Read-only file system" al crear puntos de montaje

**Hora:** (registrar al hacer commit)

**Síntoma**: en un equipo donde el rol ya se ejecutó una vez, vuelve a fallar:

```
TASK [clienteNAS : Crear los puntos de montaje locales]
[ERROR]: Task failed: Module failed: [Errno 30] Read-only file system: b'/mnt/nasDepInfo/PruebaRapidosX3'
```

**Causa raíz**: en la primera ejecución el export NFS queda montado en su punto de
destino con opción `ro` (solo lectura). En la segunda ejecución, la tarea "Crear los
puntos de montaje locales" usa el módulo `file` con `state: directory` +
`owner/group/mode`. El módulo `file` **siempre** intenta aplicar propietario y permisos
sobre el directorio destino; como sobre ese directorio hay un NFS montado `ro`, el
`chmod`/`chown` cae sobre el filesystem de solo lectura → `[Errno 30] Read-only file
system`. El directorio ya existía (si está montado, existe), así que no había nada que
crear: el error era puramente por intentar fijar permisos sobre el montaje RO.

**Fix** (`roles/clienteNAS/tasks/main.yml`): se sustituye la tarea única por dos:
1. **Detectar qué puntos ya están montados** con `mountpoint -q` (`changed_when: false`,
   `failed_when: false`).
2. **Crear los puntos que aún no están montados** (`when: item.rc != 0`), iterando sobre
   `nas_mp_check.results`. Los puntos ya montados se omiten, evitando el `chmod`/`chown`
   sobre el NFS RO.

Resultado: el rol es idempotente en re-ejecuciones sobre equipos donde el NAS ya está
montado.
