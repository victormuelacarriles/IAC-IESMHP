# Registro de cambios — 2026-05-03

## Problema 10: Auto-login persiste tras todos los fixes del chroot (v22.9)

### Síntoma
El sistema instalado arranca directamente al escritorio sin mostrar pantalla de login de GDM,
aunque el chroot (v22.9) confirmaba `AutomaticLoginEnable=False` en `/etc/gdm3/custom.conf`.

### Análisis del log (2-SetupSOdesdeLiveCD.sh v22.9)

El log confirmaba que PASO 9 se ejecutó correctamente:
- `AccountsService/users/` estaba **vacío** (el squashfs no incluye ese archivo — se crea en runtime)
- `custom.conf` se sobrescribió con `AutomaticLoginEnable=False` ✓
- `gnome-initial-setup-done` creado ✓
- Screensaver GDM deshabilitado ✓

Sin embargo el auto-login persistía. Hipótesis identificadas:

1. **`dpkg --configure -a`** en `3-SetupPrimerInicio.sh` (línea 95): el postinst de gdm3
   puede ejecutarse y leer los valores debconf del Live CD para regenerar `custom.conf`
   con `AutomaticLoginEnable=true`. El squashfs tiene valores debconf del Live CD configurados
   para auto-login de `ubuntu`.

2. **`apt-get full-upgrade`** sin `--force-confold`: si gdm3 se actualiza, su postinst
   puede reemplazar `custom.conf` con la versión del paquete (que en Ubuntu 26.04 tiene
   auto-login habilitado para la instalación por defecto).

3. **squashfs con auto-login**: el `rsync` de `1-SetupLiveCD.sh` copia el `custom.conf`
   del squashfs con `AutomaticLoginEnable=True`. Aunque el chroot lo fija, si algo
   interfiere antes o durante, el auto-login puede estar activo.

### Fixes aplicados (v22.10-20260503)

#### `0a-CreaISO.sh` — GDM fix en squashfs
Añadido en `customize_squashfs()` antes del `mksquashfs`:
- Se sobreescribe `/etc/gdm3/custom.conf` en el squashfs con `AutomaticLoginEnable=false`
- Esto garantiza que el `rsync` ya copia una config sin auto-login

#### `2-SetupSOdesdeLiveCD.sh` — Diagnósticos + debconf + `false` minúsculas
- Cambiado `False` → `false` en el valor del GDM config (compatibilidad)
- Añadido diagnóstico completo: muestra todos los ficheros en `/etc/gdm3/` con contenido,
  ficheros PAM `gdm*`, y estado debconf de gdm3
- Añadido `debconf-set-selections` para pre-configurar gdm3 con auto-login deshabilitado
  antes de que el postinst pueda leerlo

#### `3-SetupPrimerInicio.sh` — DEBIAN_FRONTEND + --force-confold + re-apply post-upgrade
- Añadido `export DEBIAN_FRONTEND=noninteractive` antes de `dpkg --configure -a`
- Añadido `debconf-set-selections` para gdm3 antes de `dpkg --configure -a`
- Añadido `-o Dpkg::Options::="--force-confold"` a `dpkg --configure -a`, `apt-get install` y `apt-get full-upgrade`
- Añadido bloque post-upgrade que sobreescribe `/etc/gdm3/custom.conf` después del upgrade

### Diagnóstico adicional añadido
El nuevo log de PASO 9 mostrará:
- Contenido de cada fichero en `/etc/gdm3/`
- Contenido de ficheros PAM `gdm*`
- Estado debconf de gdm3

Esto permitirá identificar exactamente qué causa el auto-login si vuelve a persistir.
