# 2026-06-11 — Cambios

## 08:38 — `NombreIP.sh`: eliminar residuo Mint que creaba `/opt/iesmhpMint`

**Síntoma**: tras instalar con la ISO, en el equipo aparecen DOS carpetas en
`/opt`: la correcta `/opt/IAC-IESMHP` (clon del repo) y una residual
`/opt/iesmhpMint` (también `/var/log/iesmhpMint`).

**Causa**: `Ubuntu/ISO/26.04/utiles/NombreIP.sh` (ejecutado por
`3-SetupPrimerInicio.sh` en el primer arranque) conservaba la cabecera de la
versión Mint:

```bash
DISTRO="Mint"
RAIZSCRIPTS="/opt/iesmhp$DISTRO"        # → /opt/iesmhpMint
RAIZLOGS="/var/log/iesmhp$DISTRO"       # → /var/log/iesmhpMint
mkdir -p "$RAIZSCRIPTS" "$RAIZLOGS"
```

Además descargaba `macs.csv` dentro de `/opt/iesmhpMint`.

**Fix** (`Ubuntu/ISO/26.04/utiles/NombreIP.sh`):
- Variables alineadas con el resto de scripts: `REPO="IAC-IESMHP"`,
  `DISTRO="Ubuntu"`, `RAIZSCRIPTS="/opt/$REPO"`,
  `RAIZLOGS="/var/log/$REPO/$DISTRO"`.
- `mkdir -p` solo de `$RAIZLOGS` (el repo en `/opt/IAC-IESMHP` ya existe).
- `LOCAL_MACS` pasa de `$RAIZSCRIPTS/macs.csv` a `$RAIZLOGS/macs.csv`
  (`/var/log/IAC-IESMHP/Ubuntu/macs.csv`). **Importante**: NO se apunta al
  `macs.csv` del repo clonado porque el script, tras encontrar la MAC,
  sobreescribe ese fichero dejando solo la línea encontrada — hacerlo sobre
  el clon ensuciaría el working tree de git.
- Marca de versión del script actualizada a `11/6/2026`.

**No tocado (anotado para futuro)**: el valor por defecto
`EQUIPOENMACS="mint"` (línea ~45) sigue siendo residuo Mint — si la MAC no
está en `macs.csv`, el equipo se renombraría a `mint`. No afecta a la carpeta
de `/opt`; valorar cambiarlo a `ubuntu` o a no renombrar si no hay match.

**Limpieza en equipos ya instalados**:
```bash
sudo rm -rf /opt/iesmhpMint /var/log/iesmhpMint
```
