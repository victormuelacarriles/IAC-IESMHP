# Registro de cambios — 2026-04-30

## Problema: falso positivo "casper está instalado" en 4-Comprobaciones.sh

### Síntoma
`4-Comprobaciones.sh` reportaba `1 error(es)` al final de la instalación:
```
[ERR] casper está instalado (debe eliminarse antes de update-initramfs)
```
A pesar de que la sección 1 del mismo script confirmaba:
```
[OK ]   Hooks casper: no presentes (correcto para sistema instalado)
```

### Causa raíz
El check en la sección 5 usaba `dpkg -l casper | grep "^ii"` para detectar casper.
`2-SetupSOdesdeLiveCD.sh` elimina los ficheros de hooks con `rm -rf` sin pasar por
`apt-get remove` (para evitar que los triggers dpkg bloqueen el chroot). El resultado
es que la base de datos dpkg sigue marcando casper como instalado (`ii`), aunque los
hooks no existen en disco y el initramfs se generó sin ellos.

### Solución aplicada (Opción A)
**Archivo:** `Ubuntu/ISO/26.04/4-Comprobaciones.sh` — sección 5, comprobación de casper.

Se sustituyó el check `dpkg -l casper` por una búsqueda de ficheros de hooks en disco:
- `find /usr/share/initramfs-tools /etc/initramfs-tools -name '*casper*'`
- Si hay hooks en disco → `[ERR]` (riesgo real)
- Si no hay hooks → `[OK]` (correcto, aunque dpkg diga que está instalado)
- Si dpkg marca casper como instalado pero hooks ausentes → `[INF]` informativo

El check de `ubiquity` se mantuvo via dpkg (ubiquity no debería estar en ningún sistema instalado legítimo).
