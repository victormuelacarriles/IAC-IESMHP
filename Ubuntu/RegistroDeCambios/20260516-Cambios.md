# Cambios 2026-05-16

## Rol Ansible `nvidia` — adaptado al soporte nativo de Ubuntu Desktop 26.04

### Hora: ~12:00

**Motivación**: el despliegue es para **Ubuntu Desktop 26.04**, que incorpora
**soporte nativo de drivers NVIDIA** vía `ubuntu-drivers-common` (selección del
driver *recomendado* para la GPU). El rol fijaba a fuego `nvidia-driver-535`
(serie antigua, anterior a 26.04) en `defaults/main.yml`, lo que instala un
driver desactualizado y posiblemente incompatible con el kernel de 26.04.

### Cambios aplicados

1. **`roles/nvidia/defaults/main.yml`**:
   - `required_nvidia_driver: "535"` → `required_nvidia_driver: ""`.
     Vacío = usar el driver **recomendado** por `ubuntu-drivers` (nativo 26.04).
     Una serie concreta (`"570"`, …) solo pinea manualmente si hace falta.
   - Eliminada `required_kernel`: la comprobación de kernel estaba comentada y
     Ubuntu 26.04 gestiona la compatibilidad kernel/driver vía DKMS.

2. **`roles/nvidia/tasks/main.yml`** reescrito:
   - Instala `ubuntu-drivers-common` (gestor nativo de drivers de Ubuntu).
   - **Idempotencia**: `nvidia-smi` previo (`failed_when: false`); si el driver
     ya está operativo no reinstala ni reinicia.
   - Si no está operativo:
     - `required_nvidia_driver == ""` → `ubuntu-drivers install` (recomendado).
     - `required_nvidia_driver == "NNN"` → `apt: nvidia-driver-NNN` (pin).
   - `reboot` **solo** si se acaba de instalar el driver (antes reiniciaba
     siempre que `apt` marcara `changed`).
   - Verificación post-instalación tolerante (`failed_when: false`) y
     comparación con el driver recomendado por `ubuntu-drivers devices`
     (solo informa, no actualiza).

3. **`roles/nvidia/CLAUDE.md`** actualizado para reflejar el nuevo flujo y la
   nueva variable.

### Notas

- Comportamiento por defecto ahora: instalar el driver NVIDIA **recomendado por
  Ubuntu 26.04**, sin pinear versión.
- En equipos sin GPU NVIDIA el rol sigue siendo un no-op (la mayoría de VMs de
  prueba).
- `test_nvidia.yml` (playbook suelto de prueba) no requiere cambios: sigue
  aplicando el rol; ahora usará el driver recomendado por defecto.

## Rol Ansible `xrdp` → `rdp` — RDP nativo de GNOME en Ubuntu Desktop 26.04

### Hora: ~13:30

**Motivación**: el antiguo rol `xrdp` solo instalaba el paquete `xrdp` y no
configuraba nada. En **Ubuntu Desktop 26.04** ya no tiene sentido: GNOME trae un
**servidor RDP nativo** (`gnome-remote-desktop`, gestionado con `grdctl --system`).
Se renombra el rol a `rdp` y se le da contenido real (deja el equipo accesible
por RDP de forma desatendida).

### Cambios aplicados

1. **Nuevo rol `roles/rdp/`** (sustituye a `roles/xrdp/`, eliminado):
   - `tasks/main.yml`:
     - Instala `gnome-remote-desktop` (`update_cache: false`, convención del
       proyecto).
     - Crea `rdp_tls_dir` (`/etc/gnome-remote-desktop`).
     - Genera el certificado TLS autofirmado con `openssl` **solo si no existe**
       (`creates: tls.crt`) → idempotente.
     - Fija propietario `gnome-remote-desktop:gnome-remote-desktop` y modo `0600`
       a `tls.key` / `tls.crt` (vía módulo `file`, idempotente).
     - Lee `grdctl --system status` y, **solo si el RDP no está ya habilitado con
       las rutas TLS correctas**, ejecuta `set-tls-key` / `set-tls-cert` /
       `set-credentials` (`no_log`) / `enable`.
     - Asegura `gnome-remote-desktop.service` (instancia *system*) `started` +
       `enabled`.
   - `handlers/main.yml`: `Reiniciar gnome-remote-desktop` (solo se dispara si se
     generó el certificado o se reconfiguró el RDP; evita reiniciar en cada pase).
   - `defaults/main.yml`: `rdp_username`/`rdp_password` (`ubuntu`/`ubuntu`),
     `rdp_tls_dir`, `rdp_cert_days` (3650), `rdp_cert_subject`.
   - `CLAUDE.md` propio del rol.

2. **`roles.yaml`**: la línea comentada `#- xrdp` pasa a `- rdp` (rol **activo**,
   se ejecuta tras `obs` y antes de `vscode`).

3. **Cadena de `CLAUDE.md` actualizada**: `CLAUDE.md` (raíz), `Ubuntu/CLAUDE.md`
   y `Ubuntu/ansible/CLAUDE.md` (tabla de roles: fila `xrdp` ⛔ → `rdp` ✅).

### Notas

- La instalación de `gnome-remote-desktop` se mantiene aunque en Ubuntu 26.04
  Desktop suele venir ya instalado: garantiza `grdctl` y el servicio *system*.
- Credenciales por defecto `ubuntu`/`ubuntu`; cambiables por inventario, `-e` o
  `vars:` en `roles.yaml`.
- Es la instancia **system** del servidor (`grdctl --system`), independiente de
  que haya sesión gráfica iniciada — adecuado para el primer arranque desatendido.
