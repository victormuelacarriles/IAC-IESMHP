# Cambios 2026-05-15

## 2-SetupSOdesdeLiveCD.sh — v22.17-20260515

### Hora: ~10:30

**Motivación**: tras instalar el SO y arrancar por primera vez, el usuario `usuario` puede iniciar sesión correctamente, pero al abrir un terminal y ejecutar `sudo apt update` se obtiene el siguiente error:

```
Err:2 file:/cdrom resolute Release
  Fichero no encontrado - /cdrom/dists/resolute/Release (2: No existe el archivo o el directorio)
Error: El repositorio «file:/cdrom resolute Release» no tiene un fichero de Publicación.
Información: No se puede actualizar de un repositorio como este de forma segura y por tanto está deshabilitado por omisión.
```

### Diagnóstico

El Live CD de Ubuntu añade automáticamente una entrada APT que apunta al ISO montado en `/cdrom` (entrada `deb cdrom:[Ubuntu ...]/ resolute main`). Esa entrada vive en `/etc/apt/sources.list` (formato clásico) o en `/etc/apt/sources.list.d/*.sources` (DEB822, Ubuntu 26.04).

El rsync de `1-SetupLiveCD.sh` copia esa configuración tal cual al sistema instalado. Tras el reboot, el sistema instalado ya no tiene `/cdrom` montado, así que cualquier `apt update` falla con el error visto. El sistema sigue funcionando porque las otras fuentes (archive.ubuntu.com, security.ubuntu.com) sí responden, pero el aviso es ruidoso y entorpece a los usuarios.

### Cambios aplicados

1. **Nuevo paso "Limpiar fuentes APT del Live CD (cdrom:)"** en `2-SetupSOdesdeLiveCD.sh`, insertado tras la generación de `/etc/fstab` y antes de "Verificar conectividad a Internet". Se ejecuta dentro del chroot, sobre las fuentes APT del sistema **instalado**, no del Live CD.

2. **Dos capas de limpieza**:
   - **Formato clásico** (`/etc/apt/sources.list` y `/etc/apt/sources.list.d/*.list`): se eliminan con `sed` las líneas que coincidan con la regex `^[[:space:]]*deb(-src)?[[:space:]]+(\[[^]]*\][[:space:]]+)?cdrom:` (cubre `deb cdrom:`, `deb-src cdrom:` y variantes con opciones `[arch=...]`).
   - **Formato DEB822** (`/etc/apt/sources.list.d/*.sources`, Ubuntu 26.04): si el fichero contiene una línea `URIs:` con `cdrom:`, se elimina el fichero completo (un `.sources` puede tener varios stanzas, pero el cdrom típicamente está aislado en su propio fichero como `ubuntu-cdrom.sources`).

3. **Diagnóstico**: antes de borrar, se loguean las líneas/ficheros que se van a eliminar para dejar rastro en `2-SetupSOdesdeLiveCD.sh.log`. Al final del paso se reporta el número total de fuentes limpiadas; si es 0, se loguea con `[INF]` (no es un error: simplemente esa versión del Live CD no añadió cdrom).

4. **Versión** — `VERSIONSCRIPT` de `22.16-20260514` → `22.17-20260515`.

### Cómo verificar tras regenerar la ISO

Dentro del sistema instalado, tras arrancar como `usuario`:

```bash
# No debe aparecer ninguna entrada cdrom:
grep -rn 'cdrom:' /etc/apt/sources.list /etc/apt/sources.list.d/ 2>/dev/null

# 'apt update' debe completar sin el error "file:/cdrom":
sudo apt update
```

En el log de chroot (`/var/log/IAC-IESMHP/Ubuntu/2-SetupSOdesdeLiveCD.sh.log`) debe verse el paso "Limpiar fuentes APT del Live CD" con al menos un `[OK]` indicando el fichero/línea limpiada.
