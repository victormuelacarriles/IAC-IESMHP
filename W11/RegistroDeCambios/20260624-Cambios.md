# 2026-06-24 — Cadena de aprovisionamiento W11 (ISO): mejoras

Resumen de los cambios introducidos en `W11/ISO/` para que la ISO de Windows 11
deje el equipo **completamente listo** (config + software + discos + Windows
Update + compactado) de forma desatendida, con control de tiempos y logs.

> **Recordatorio de despliegue**: `comun.ps1` es un fichero **nuevo** del que
> dependen `1/2/3-*.ps1` y (tras clonar) `0b-GitHub.ps1`. Hay que hacer
> **commit + push a `main`** de todos estos ficheros **antes** de arrancar una
> ISO, porque `0b-GitHub.ps1` los clona desde GitHub. (Mismo caso que `comun.sh`
> en Ubuntu.)

## 0 — Logs al lado de cada fichero
- Cada script de la cadena escribe su log **junto a sí mismo** con el mismo
  nombre + `.log` (`0b-GitHub.ps1.log`, `1-Setup.ps1.log`, `2-Aplicaciones.ps1.log`,
  `3-Particionado.ps1.log`), todos en `C:\Program Files\IAC-IESMHP\W11\ISO\`.
- `LimpiaW11.ps1` / `CompactaW11.sh` mantienen su convención previa (log con marca
  de tiempo en el nombre, junto al script).

## 1 — `0b-GitHub.ps1`: ubicación definitiva + control de tiempos
- **Ubicación desde el principio**: `C:\Program Files\IAC-IESMHP\W11\ISO\0b-GitHub.ps1`.
  `0-CreaIsoW11.sh` lo embebe en `$OEM$\$1\Program Files\IAC-IESMHP\W11\ISO` y el
  `FirstLogonCommands` del `autounattend.xml` lo invoca desde ahí. Cuando 0b clona
  el repo, este se materializa **sobre esa misma carpeta**: clonado **in-place**
  (`git init`+`fetch`+`checkout -f`) porque `git clone` exige directorio vacío.
- **Control de tiempos** (`comun.ps1`): `Tiempos.log` acumula una línea por fase
  (`inicio|fin|duración`) y un **TOTAL** desde **T0 = instalación de Windows**
  (`Win32_OperatingSystem.InstallDate`). 0b anota la fase `github (clonado)`.

## 2 — `1-Setup.ps1`: orquestador
- Se ejecuta en **ventana PowerShell visible** (lo lanza 0b con `Start-Process
  -WindowStyle Normal`).
- **Energía "Alto rendimiento"** imitando el rol Ansible `energiaAltoRendimiento`
  (`powercfg /setactive 8c5e7fda-…` + `standby-timeout-ac/dc 0`).
- **Pantalla de inicio de sesión** al **finalizar** la configuración: se desactiva
  el autologon (que se re-arma durante Windows Update) → arranques posteriores con
  login/contraseña.
- **Windows Update completo** con `PSWindowsUpdate`, gestionando **varios
  reinicios** mediante autologon re-armado + tarea programada `IAC-IESMHP-Reanudar`
  (relanza `1-Setup.ps1 -Reanudar`) + fase en `HKLM\SOFTWARE\IAC-IESMHP`.
- **Compactado** llamando a **`W11/Utiles/Compacta/LimpiaW11.ps1`** (limpieza +
  zero-fill dentro de Windows). `CompactaW11.sh` es del host Linux y NO se ejecuta
  aquí. **Orden**: el compactado es **siempre el último** paso.

## 3 — `2-Aplicaciones.ps1` (nuevo)
- Instala el software de `W11/ansible/roles.yaml` con **winget** (preferente) +
  **chocolatey** (lo que no está bien en winget). Dos listas editables
  (Nombre/Id/Version/Args), dos bucles, **tiempo por aplicación** y **continúa si
  una falla** (resumen final con las fallidas).

## 4 — `3-Particionado.ps1` (nuevo)
- CD/DVD→`R:`, USB→`S,T…`; discos **sin formatear** clasificados **NVMe→SSD→SATA**
  (mayor→menor), formateados **NTFS** con letras **`D,E,F…`**; perfiles por defecto
  en **`D:\Users`**. **Nunca** toca el disco de sistema ni USB; inocuo en VM de un
  solo disco.
